# План реализации кнопок AI-обработки

На основании [project.md](./project.md).

## Цель

Добавить функции для трёх кнопок:

- **«О чем статья?»** — краткое описание содержания статьи на русском
- **«Тезисы»** — ключевые тезисы и выводы на русском
- **«Пост для Telegram»** — готовый пост для публикации в Telegram на русском

Общий сценарий для каждой кнопки: **URL → парсинг HTML → запрос к OpenRouter (Deepseek) → вывод результата**.

## Порядок действий

### Шаг 1. Вынести общую логику подготовки статьи

**Файл:** `lib/article-text.ts` (новый)

1. Создать функцию `prepareArticleText(title, content)`, которая:
   - проверяет наличие `content`;
   - обрезает текст до лимита (например, 12 000 символов);
   - формирует единый текст для промпта: заголовок + содержимое.
2. Использовать эту функцию в `translateArticle` и во всех новых AI-функциях, чтобы не дублировать код.

---

### Шаг 2. Добавить AI-функции с промптами

**Файл:** `lib/openrouter.ts` (расширить)

Для каждой кнопки — отдельная функция, использующая `chatCompletion`:

| Функция | Назначение | System prompt (суть) |
|---------|------------|----------------------|
| `summarizeArticle(title, content)` | «О чем статья?» | Кратко опиши на русском, о чём статья (2–4 абзаца), без воды |
| `extractTheses(title, content)` | «Тезисы» | Выдели 5–10 ключевых тезисов на русском в виде маркированного списка |
| `generateTelegramPost(title, content)` | «Пост для Telegram» | Напиши пост для Telegram: цепляющий заголовок, суть, призыв; до ~1500 символов, без markdown-таблиц |

**Порядок реализации функций:**

1. `summarizeArticle` — самая простая, хорошая точка для проверки пайплайна
2. `extractTheses` — похожая структура, другой формат ответа
3. `generateTelegramPost` — отдельные требования к стилю и длине

---

### Шаг 3. Создать API-маршруты

По аналогии с `app/api/translate/route.ts` — три новых endpoint:

| Маршрут | Функция | Ответ |
|---------|---------|-------|
| `POST /api/summary` | `summarizeArticle` | `{ "result": "..." }` |
| `POST /api/theses` | `extractTheses` | `{ "result": "..." }` |
| `POST /api/telegram` | `generateTelegramPost` | `{ "result": "..." }` |

**Для каждого маршрута:**

1. Принять `{ "url": "..." }` в теле запроса
2. Валидировать URL (как в `/api/translate`)
3. Вызвать `fetchAndParseArticle(url)`
4. Вызвать соответствующую AI-функцию
5. Вернуть JSON с результатом или ошибкой
6. Указать `export const runtime = "nodejs"` и `export const dynamic = "force-dynamic"`

**Альтернатива (опционально, позже):** один маршрут `POST /api/generate` с параметром `{ url, action: "summary" | "theses" | "telegram" }` — меньше дублирования, но менее явно.

---

### Шаг 4. Подключить кнопки в интерфейсе

**Файл:** `app/components/ReferentApp.tsx`

1. В `handleAction` для каждого действия вызывать свой API:

   | Action | Endpoint |
   |--------|----------|
   | `summary` | `/api/summary` |
   | `theses` | `/api/theses` |
   | `telegram` | `/api/telegram` |

2. Обновить `LOADING_LABELS`:
   - `summary` → «Анализ статьи…»
   - `theses` → «Формирование тезисов…»
   - `telegram` → «Создание поста…`

3. Выводить `data.result` в блок «Результат» (как текст, не JSON).

4. Использовать `readJsonResponse` для безопасного разбора ответа.

5. Убрать текущую заглушку с выводом JSON из `/api/parse` для этих трёх кнопок.

---

### Шаг 5. Проверка и отладка

1. **Локальный запуск:** `pnpm dev`, открыть адрес из терминала
2. **Тестовые URL:** короткая статья (`https://example.com`) и реальная англоязычная статья
3. **Проверить каждую кнопку:**
   - «О чем статья?» — связный краткий пересказ на русском
   - «Тезисы» — список пунктов, без повторов
   - «Пост для Telegram» — читаемый пост подходящей длины
4. **Проверить ошибки:**
   - пустой URL
   - некорректный URL
   - страница без контента
   - отсутствие `OPENROUTER_API_KEY`
   - ошибки OpenRouter (credits, rate limit)
5. **Сборка:** `pnpm build` — убедиться, что нет ошибок TypeScript

## Переменные окружения

Уже используются, менять не нужно:

```env
OPENROUTER_API_KEY=sk-or-v1-...
OPENAI_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_MODEL=openrouter/free
```

Модель по умолчанию: `openrouter/free` (бесплатная). Можно указать конкретную free-модель, например `google/gemma-4-31b-it:free`.
